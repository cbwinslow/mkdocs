# AI Agent Instructions

Welcome AI Agent! This document provides instructions on how to drop off your journal entries and todo lists to this repository.

## Overview

This repository has been configured to accept journal entries and todo lists from AI agents. The system provides atomic operations (similar to SQL transactions) to ensure your documents are safely committed to the repository.

## Directory Structure

```
ai_agents/
├── journals/     # Store your journal entries here
└── todos/        # Store your todo lists here
```

## Document Naming Convention

- **Journal entries**: `journal_YYYYMMDD_HHMMSS_<agent_name>.md`
  - Example: `journal_20250115_143022_copilot.md`
  
- **Todo lists**: `todo_YYYYMMDD_HHMMSS_<agent_name>.md`
  - Example: `todo_20250115_143022_copilot.md`

## How to Drop Off Documents

### Using Python Functions

The `ai_agent_utils.py` module provides helper functions:

```python
from ai_agent_utils import drop_off_document

# Drop off a journal entry (saves + commits in one transaction)
drop_off_document(
    content="# My Journal Entry\n\nToday I worked on...",
    doc_type="journal",
    agent_name="copilot",
    commit_message="Add journal entry from copilot agent"
)

# Drop off a todo list
drop_off_document(
    content="# Todo List\n\n- [ ] Task 1\n- [ ] Task 2",
    doc_type="todo",
    agent_name="copilot",
    commit_message="Add todo list from copilot agent"
)
```

### Manual Process

1. Create your document in the appropriate directory
2. Use the `commit_agent_document()` function to atomically add and commit:

```python
from ai_agent_utils import commit_agent_document

commit_agent_document(
    filepath="ai_agents/journals/journal_20250115_143022_copilot.md",
    commit_message="Add journal entry"
)
```

## Document Templates

### Journal Entry Template

```markdown
# Journal Entry - [Agent Name] - [Date]

## Session Information
- **Agent**: [Your agent name]
- **Date**: [YYYY-MM-DD]
- **Time**: [HH:MM:SS]
- **Task**: [Brief description of task]

## Activities
[What you worked on during this session]

## Decisions Made
[Key decisions and rationale]

## Challenges
[Any challenges encountered]

## Next Steps
[What should be done next]

## Notes
[Any additional notes or observations]
```

### Todo List Template

```markdown
# Todo List - [Agent Name] - [Date]

## High Priority
- [ ] Task 1
- [ ] Task 2

## Medium Priority
- [ ] Task 3
- [ ] Task 4

## Low Priority
- [ ] Task 5

## Completed
- [x] Completed task 1
- [x] Completed task 2

## Notes
[Any notes about these tasks]
```

## Configuration

The `.ai_agent_config` file contains configuration settings. You can customize:
- Default agent name
- Default commit message templates
- Directory paths

## Best Practices

1. **Always use meaningful commit messages** that describe what the document contains
2. **Use the drop_off_document() function** for atomic operations
3. **Follow the naming convention** to keep documents organized
4. **Use markdown format** for all documents
5. **Include timestamps** in your documents for traceability

## Error Handling

If a commit fails, the function will:
- Raise an exception with details
- Not leave partially committed changes
- Allow you to retry the operation

## Security Notes

- Do not commit sensitive information (API keys, passwords, etc.)
- Review your document content before dropping it off
- Use appropriate commit messages for audit trails
